In the search image page, there's an input that allows us to search for an image by its ID.
The first intuition is to look for an SQL injection.

If we input something that is not an id, the site doesn't display any error or entry.
This could tell us that there's no injection possible, but if we try to input a potential SQL injection like "1 OR 1=1", the server does return every entry!
From this, we know that even if the site doesn't directly show an error, we can inject our own input into the SQL request.

We will use something called a union-based injection. Using the UNION keyword, we can return results from another table.
The first step is then to get information on the database architecture.
    1 OR 1=1 UNION SELECT NULL, table_name FROM information_schema.tables WHERE table_schema = database()

"SELECT NULL, table_name" allows us to return any amount of columns we want.
Normally, the UNION keyword only works if the left and right sections of the request return the same number of columns.
Since the first request returns "Title" and "Url", we can't only return table_name: we need to add a NULL field.

"FROM information_schema.tables WHERE table_schema = database()" queries for the tables names in the database.
If we didn't have the WHERE, we would query for all the tables in the whole server, including internal ones that would pollute our result.

Thanks to this query, we find a table named "list_images". We will then list the columns in this table using the same method.
    1 OR 1=1 UNION SELECT NULL, column_name FROM information_schema.columns WHERE table_name = CHAR(108,105,115,116,95,105,109,97,103,101,115)

We can't directly write "list_images" because the server sanitizes user-inputted quotes.
Still, string literals must be quoted in SQL, thus we use CHAR() which returns a quoted string from ASCII decimals.
This finally gives us all columns present in the users table:
    id,url,title,comment

One field is suspicious: comment.
Our first instinct is then to query for that value for each image.
    1 OR 1=1 UNION SELECT NULL, comment FROM list_images

For the last image, this gives us:
    If you read this just use this md5 decode lowercase then sha256 to win this flag ! : 1928e8083cf461a51303633093573c46

This is an MD5 password, which is easily crackable. The password is "albatroz", then sh256-ed gives the flag:
